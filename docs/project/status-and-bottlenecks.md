## 🎮 瓶颈 #1：核心玩法机制（Critical）

### 问题描述

> **核心矛盾**：我们有了完整的剧情和世界观，但"这个游戏要怎么玩？"

目前的设定已经非常丰富：
- ✅ 影子共生系统（依附/游离/消逝）
- ✅ 情绪投射机制（恐惧→怪物、孤独→迷宫）
- ✅ 睡眠阶段系统（清醒期→浅睡期→深睡期→REM期）
- ✅ 医学元素设计（疾病→异常状态、病原→怪物、心理→技能）

**但核心玩法循环仍未确定**，这直接导致：
- Agent A 无法确定程序需求文档的核心系统规格
- Agent C 无法确定美术资源的动作需求（是否需要战斗动画？）
- 无法确定音效需求（战斗音效？环境音效？）
- 整个第二阶段 Agent 协调工作无法推进

### 待决策问题清单

#### 1.1 核心体验类型（必须首先确定）

**问题**：玩家在游戏中主要做什么？

- [ ] **选项A：动作战斗导向**
  - 实时战斗，技能释放，成长build
  - 参考：Hades、以撒的结合、元气骑士
  - 优势：玩法深度高，重复可玩性强
  - 挑战：需要设计影子无实体的战斗方式

- [ ] **选项B：探索解谜导向**
  - 环境解谜，记忆收集，剧情推进
  - 参考：Gris、Journey、Inside
  - 优势：与叙事和氛围契合度高
  - 挑战：需要设计有深度的解谜机制

- [ ] **选项C：叙事体验导向**
  - 步行模拟器，互动叙事，情感体验
  - 参考：What Remains of Edith Finch、Dear Esther
  - 优势：最大化剧情和情感冲击力
  - 挑战：游戏性较弱，可能受众有限

- [ ] **选项D：混合机制**（推荐考虑）
  - 探索解谜为主 + 轻度战斗/潜行
  - 参考：Hollow Knight、Ori
  - 优势：平衡玩法深度和叙事体验
  - 挑战：需要精心设计两种机制的融合

#### 1.2 战斗系统（如果选择包含战斗）

**问题**：如果要战斗，怎么战斗？

**战斗类型**：
- [ ] 实时动作战斗（类Hades）
- [ ] 回合制战斗（类JRPG）
- [ ] 混合机制（实时移动+回合技能）
- [ ] 其他创新机制

**操作方式**：
- [ ] 键盘/手柄操作方案
- [ ] 技能释放方式（按键组合？技能轮盘？）
- [ ] 闪避/防御机制（影子如何闪避？）

**战斗节奏**：
- [ ] 快节奏爽快型（高伤害，快速击杀）
- [ ] 慢节奏策略型（需要观察，寻找破绽）
- [ ] 随睡眠阶段变化（浅睡期慢，REM期快）

#### 1.3 影子战斗方式（世界观一致性）

**问题**：影子没有实体，如何与敌人互动？

- [ ] **方案A：依附物体战斗**
  - 影子依附于梦境中的物体（武器、道具）进行战斗
  - 优势：符合影子特性，有创意
  - 挑战：需要设计物体获取和切换机制

- [ ] **方案B：情绪能量攻击**
  - 影子使用情绪能量进行远程/范围攻击
  - 优势：与情绪投射机制结合
  - 挑战：需要设计能量管理系统

- [ ] **方案C：环境互动**
  - 利用梦境环境（光影、记忆碎片）进行"战斗"
  - 优势：独特，符合梦境主题
  - 挑战：设计难度大，可能不够直观

- [ ] **方案D：概念具象化**
  - 影子在梦境中可以暂时"实体化"
  - 优势：简化设计，传统战斗方式
  - 挑战：与影子无实体的设定有冲突

#### 1.4 睡眠阶段与玩法结合

**问题**：如何将睡眠阶段系统融入核心玩法？

| 睡眠阶段 | 当前设定机制 | 待确定：玩法表现 |
|---------|-------------|-----------------|
| 清醒期 | 路线选择、装备整备 | 是策略规划还是可玩内容？ |
| 浅睡期 | 状态波动、场景重组 | 如何利用不稳定性设计玩法？ |
| 深睡期 | 恐惧累积、资源稀缺 | 是生存挑战还是恐怖体验？ |
| REM期 | 战斗加成、BOSS战 | 如何设计高潮体验？ |

#### 参考游戏深度分析

| 游戏 | 核心机制 | 与《影梦》的契合点 | 可借鉴之处 |

| 因素 | 战斗导向 | 解谜导向 | 叙事导向 |

## 📖 标准化工作流程

### 瓶颈处理流程

当本文档中的瓶颈得到解决时，按以下步骤处理：

```
步骤1: 标记状态
├── 在优先级总览表中将该项标记为"✅ 已确定"或"✅ 已解决"
└── 添加决策结果简述

步骤2: 样式处理
├── 在文档主体中，为该瓶颈标题添加删除线（~~标题~~）
├── 在表格中添加删除线标记
└── 使用灰色/淡化效果降低视觉突出度

步骤3: 信息归档
├── 将已解决的瓶颈完整内容复制到"已解决的瓶颈（归档区）"
├── 保留关键决策信息（日期、结果、负责人）
├── 添加关联文档链接
└── 可选：添加决策背景简述

步骤4: 文档清理
├── 从优先级总览表中移除或降级该项
├── 更新待办清单，移除相关任务
└── 更新决策记录表

步骤5: 更新日志
├── 在"文档更新日志"中记录本次变更
├── 注明变更类型："已解决并归档"
└── 记录变更者和日期

步骤6: 通知相关方
├── 通知所有Agent该瓶颈已解决
├── 提供关联文档链接
└── 说明对下游工作的影响
```
