# GAME_MAKING
这是一个用于游戏开发的仓库，存储一些游戏开发相关代码

# 像素肉鸽动作游戏项目总结

## 项目概述

- **游戏类型**：俯视角伪像素风格肉鸽动作游戏
- **参考游戏**：《元气骑士》《喵喵启示录》《饥荒》
- **核心循环**：局内失败进度清零 + 局外成长永久保留
- **美术风格**：像素风格
- **游戏引擎**：Godot 4.x
- **开发状态**：概念设计阶段，故事剧情待定

## 核心玩法设计

### 1. 基础玩法
- 俯视角动作射击，伪3D表现（2D精灵 + 透视/缩放模拟深度）
- 房间清版式推进，随机生成关卡
- 局内成长：临时武器/技能，失败后清零
- 局外成长：永久解锁角色能力、武器、升级

### 2. 特色系统
- **Boss武器夺取**：击败或收服小Boss后获取其武器或技能（有CD）
- **攻击方式多样化**：设计丰富武器类型（近战、远程、范围、特殊效果）
- **武器组合与升级**：每种武器有多个升级分支，形成不同流派

### 3. 进度系统
- **局内资源**：临时金币、弹药、血量恢复（失败清零）
- **局外资源**：永久货币、解锁点数、收藏品（永久保留）
- **成长树**：角色属性、武器解锁、被动技能升级

## 技术实现方案

### 引擎与工具
- **主引擎**：Godot 4.x（2D引擎对像素游戏支持优秀）
- **美术工具**：Aseprite / Pixelorama（像素动画）
- **版本控制**：Git（GitHub/Gitee）
- **项目管理**：Trello / GitHub Projects + 飞书协作

### 关键实现要点
1. **像素美术设置**：纹理导入Filter → Nearest，关闭Mipmaps，启用像素对齐
2. **场景结构**：节点化设计，分离UI层、世界层、实体层
3. **房间生成**：预置房间模板 + 随机连接算法
4. **武器系统**：基类继承模式，支持热插拔切换
5. **数据持久化**：Resource格式保存永久进度，Session数据临时存储

## 16周开发规划

### 阶段1：概念设计（第1-2周）
- 确定故事世界观（一页纸梗概）
- 定义核心玩法循环流程图
- 确定美术风格参考（情绪板）
- 列出MVP功能清单（不超过10个核心功能）
- 建立开发环境与Git仓库

### 阶段2：预生产（第3-4周）
- 创建玩家角色像素动画（4方向行走/攻击）
- 实现玩家移动与摄像机跟随
- 设计基础敌人（1-2种）及行为
- 设计第一个武器（近战/远程）
- 制作简单关卡（一个房间模板）

### 阶段3：核心系统开发（第5-8周）
- 武器系统（基类、切换、升级）
- 敌人AI状态机（巡逻、追击、攻击）
- 房间生成系统（随机连接3-5个房间）
- UI框架（血条、弹药、小地图）
- 音效系统（背景音乐、攻击音效）

### 阶段4：内容生产（第9-12周）
- 制作5-8种敌人（含1个Boss）
- 制作10-15种武器/技能
- 设计10-20个房间模板
- 制作全套UI（菜单、设置、商店）
- 实现局外成长系统

### 阶段5：测试与优化（第13-14周）
- 平衡性测试（难度曲线调整）
- Bug排查与修复
- 性能优化（内存、帧率）
- 小范围玩家测试反馈
- 本地化与配置支持

### 阶段6：发布准备（第15-16周）
- 多平台打包测试（Windows/macOS/Linux）
- 商店页面素材准备（截图、视频、图标）
- 撰写商店描述与更新日志
- 最终测试与签名
- 发布上线与后续计划

## 故事剧情框架（建议）

**暂定标题**：《豆豆核心：像素深渊》

**世界观**：
在一个由“像素能量”构成的数字世界里，一股“数据腐化”正在蔓延，将原本和谐的像素生物变成狂暴的敌人。你作为“豆豆人守护者”，必须深入不断变化的“深渊迷宫”，击败腐化首领，收集失落的“核心碎片”，恢复世界平衡。

**核心设定**：
- **随机生成解释**：数据腐化导致世界结构不稳定
- **敌人多样性**：腐化能量重组不同像素生物
- **成长逻辑**：核心碎片蕴含原始像素能量，可永久强化守护者
- **扩展性**：可引入不同腐化区域（冰原、火山、机械废墟等）
