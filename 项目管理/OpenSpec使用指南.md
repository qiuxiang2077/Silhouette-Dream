# Trae AI Skills 使用指南

> **读这一篇就够了**：用大白话告诉你 `.trae/skills` 是什么，以及怎么用它来高效开发游戏。

---

## 一、什么是 `.trae/skills`？

**`.trae/skills`** 是 Trae AI 的「技能包」文件夹，里面放着各种 AI 技能（Skill）。

**类比理解**：
- 你可以把 Trae AI 想象成一个刚入职的程序员
- `.trae/skills` 就是他的「工作手册」，告诉他遇到什么问题该怎么做
- 每个 skill 文件（比如 `openspec-propose`）就是一张「执行地图」，AI 会按这个地图的步骤来工作

**你们项目的 skill 目前有**：
| Skill 名称 | 作用 |
|-----------|------|
| `openspec-propose` | **提想法** — 把你的需求变成完整的开发计划 |
| `openspec-apply-change` | **做开发** — 按计划一步步实现功能 |
| `openspec-archive-change` | **收存档** — 把完成的工作归档 |
| `openspec-explore` | **聊想法** — 先想清楚再动手 |

---

## 二、OpenSpec 是什么？

**OpenSpec = 规范驱动开发**

大白话就是：**先想清楚要做什么，再开始写代码**。

**传统开发**（容易翻车）：
```
我有个想法 → 直接开干 → 做到一半发现想错了 → 改改改 → 勉强完成
```

**OpenSpec 开发**（有条不紊）：
```
我有个想法 → AI帮你写成「计划书」→ 你确认计划 → AI按计划一步步实现 → 完成
```

**好处**：
- 不会做到一半发现方向错了
- 每个功能的需求、设计、任务都记录下来了
- 以后回头看，知道当时为什么这么做

## 三、4个技能怎么配合用？

### 场景：你想给游戏加一个新功能

**第一步：聊想法** `/opsx:explore`
```
你：「我想给游戏加一个白天/黑夜切换系统」
AI：（和你一起讨论这个想法的细节）
```

**第二步：提计划** `/opsx:propose`
```
你：「帮我创建一个昼夜切换系统的提案」
AI：自动生成一套完整文档：
   - proposal.md（做什么、为什么做）
   - design.md（怎么做）
   - tasks.md（具体任务清单）
```

**第三步：做开发** `/opsx:apply`
```
你：「开始实现」
AI：按 tasks.md 的清单，一个任务一个任务帮你做
```

**第四步：收存档** `/opsx:archive`
```
你：「完成了，帮我归档」
AI：把这次的工作整理到 archive 文件夹，备以后查阅
```

## 四、实际举例

### 场景：你想实现「玩家移动功能」

**1. 告诉 Trae AI：**
> 「帮我用 OpenSpec 创建一个玩家移动系统的提案」

**2. AI 自动生成：**
```
openspec/changes/add-player-movement/
├── proposal.md    ← 需求文档（做什么、为什么）
├── design.md      ← 设计文档（怎么做）
├── tasks.md       ← 任务清单（具体步骤）
└── specs/         ← 详细规范
```

**3. 你检查 proposal.md，确认方向没错：**
```markdown
# 玩家移动系统提案

## 做什么
实现玩家角色的基础移动功能...

## 为什么
玩家需要能够控制角色在关卡中移动...

## 验收标准
- 角色能左右移动
- 移动手感流畅
- 能跳跃
```

**4. 让 AI 开始实现：**
> 「开始实现」

AI 会按 tasks.md 一个个完成任务，每完成一个打一个勾。

**5. 完成后归档：**
> 「完成了，帮我归档」

## 五、什么时候用哪个？

| 你想做的事 | 对应的 Skill |
|-----------|-------------|
| 有一个想法，想先聊聊、理清思路 | `/opsx:explore` |
| 想好了，要开始做，先制定计划 | `/opsx:propose` |
| 已经有计划了，要开始写代码 | `/opsx:apply` |
| 功能做完了，要归档 | `/opsx:archive` |

---

## 六、注意事项

1. **先想再写** — 用 `/opsx:explore` 聊清楚再用 `/opsx:propose`
2. **看提案再动手** — AI 生成提案后，先确认方向没错再让 AI 实现
3. **不用死记硬背** — 你只需要说「帮我创建一个 XXX 功能的提案」，AI 会自动按流程走

---

## 七、快速开始

**最常用的流程**：
```
1. 告诉 AI：「帮我用 OpenSpec 创建 [功能名] 提案」

2. AI 生成提案，询问你是否开始实现

3. 你说：「开始实现」

4. AI 一个个完成任务

5. 你说：「完成了，归档」
```

## 八、文件结构

```
GAME_MAKING/
├── .trae/skills/           ← Trae AI 技能包（你不需要改这里）
│   ├── openspec-propose/   ← 提想法的技能
│   ├── openspec-apply-change/  ← 做开发的技能
│   ├── openspec-archive-change/ ← 归档的技能
│   └── openspec-explore/   ← 聊想法的技能
│
└── openspec/               ← OpenSpec 工作区（存放你的项目计划）
    ├── config.yaml         ← 配置文件
    └── changes/            ← 你的功能变更记录
        └── add-xxx/         ← 某个功能的完整提案+实现
```
