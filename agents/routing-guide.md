# Trae SoloCoder 智能路由指南

> **文档目的**：指导 Trae SoloCoder（AI助手）在何时调用哪个Agent或Skill
> 
> **使用方式**：根据用户输入的关键词和意图，自动匹配最佳处理方案

---

## 🎯 决策流程图

```
用户输入
    ↓
[分析意图类型]
    ↓
┌─────────────┬─────────────┬─────────────┬─────────────┐
│   创意整理   │   需求分析   │   代码实现   │   美术资源   │
│  (Agent D)  │  (Agent A)  │  (Agent B)  │  (Agent C)  │
└─────────────┴─────────────┴─────────────┴─────────────┘
    ↓              ↓              ↓              ↓
调用对应Agent   调用对应Agent   调用对应Agent   调用对应Agent
或 Skill       或 Skill       或 Skill       或 Skill
```

## 🔍 意图识别规则

### 规则1：关键词优先级
```
OpenSpec命令 > Agent特定关键词 > 通用技能
```

### 规则2：多意图处理
如果用户输入包含多个意图：
1. 识别主要意图（最紧急/最重要的）
2. 询问用户是否需要处理其他意图
3. 或按顺序依次处理

**示例**：
- 用户："整理一下新想法，然后设计战斗系统"
- 处理：先调用Agent D整理，再询问是否继续设计

### 规则3：模糊意图澄清
当无法确定调用哪个Agent时：
```
用户："帮我处理一下这个"
↓
Trae："您希望我从哪个角度处理？
       1. 整理创意（Agent D）
       2. 技术分析（Agent A）
       3. 代码实现（Agent B）
       4. 美术设计（Agent C）"
```

## ⚠️ 特殊情况处理

### 情况1：用户直接@某个角色
```
用户："@游戏需求分析师 分析一下这个系统"
↓
直接调用Agent A，无需判断
```

### 情况2：跨Agent协作
```
用户："整理创意并生成需求文档"
↓
Trae："这需要Agent D和Agent A协作完成。
       我先调用Agent D整理创意，
       然后调用Agent A生成需求文档。"
↓
[顺序调用多个Agent]
```

### 情况3：使用Skill
```
用户："生成游戏wiki"
↓
Trae："调用游戏Wiki生成师（docs-generator）..."
↓
[调用docs-generator skill]
```

## 🎮 与游戏项目结合

### 项目特定路由

| 项目阶段 | 主要调用 | 辅助调用 |
|---------|---------|---------|
| **第一阶段**（准备） | Agent D + docs-generator | - |
| **第二阶段**（编码） | Agent A → Agent B + Agent C | OpenSpec skills |
| **瓶颈解决** | OpenSpec explore/propose | Agent A |

### 当前项目状态判断
根据`开发瓶颈与待决策事项.md`：
- **核心玩法未确定** → 优先调用 openspec-explore
- **需要变更管理** → 调用 OpenSpec skills
- **设定需要更新** → 调用 Agent D
